# Overmind v2.1 Tier 1 — Self-Improving Audit, VeriMAP, Q-Router, Hash Loop, Tests-First

> Date: 2026-04-07
> Status: APPROVED

## 5 Features

### 1. Self-Improving Audit Loop
After each verification, compare results against project history. Track rolling pass
rate. Improvements get persisted, degradations get flagged as regressions.
- `AuditLoop.evaluate()` compares current vs last 5 snapshots
- Stores `audit_snapshot` memories per project
- CLI: `overmind audit-history --project-id X`

### 2. VeriMAP — Verify Functions in DAG
Each task gets `verify_command`. After agent completes (exit 0), run verify before
marking COMPLETED. Failures auto-requeue (max 2 retries).

### 3. Q-Learning Router
Replace hardcoded routing strengths with learned Q-values per (runner_type, task_type).
`q_value = (wins + 1) / (wins + losses + 2)` (Laplace smoothing).
Bootstrap from policies.yaml strengths.

### 4. Output-Hash Loop Detection
Hash normalized content in sliding window of 20. If any fingerprint appears 3+ times,
loop detected. Existing line-based detection stays as fast path.

### 5. Tests-First Task Template
For advanced math projects: generate 2 chained tasks (write tests → implement).
Regular projects unchanged.
