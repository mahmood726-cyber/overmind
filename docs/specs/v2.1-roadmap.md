# Overmind v2.1+ Roadmap

> Compiled 2026-04-07 from portfolio audit (312 projects) + research across
> GitHub, Medium, arXiv, and agent orchestration community.

## Priority Tier 1 — Highest ROI (v2.1)

### 1. Self-Improving Audit Loop
**Source:** AutoAgent (kevinrgu) + self-improving agent patterns
**What:** After each verification cycle, compare pass rates against previous runs.
If a prompt/strategy change improved results, persist it. If not, revert.
Over time, Overmind's prompts evolve to maximize verification success.
**Implementation:**
- Track per-project verification history in memory (pass rate, prompt hash, strategy)
- After each tick, compute delta vs. previous
- If delta > 0: promote the change (boost memory relevance, persist prompt variant)
- If delta < 0: revert (archive the change, restore previous prompt)
- Dream cycle analyzes trends and promotes stable improvements to heuristics
**Effort:** 2-3 days

### 2. VeriMAP: Verification Functions in DAG
**Source:** arXiv 2510.17109
**What:** Each task node in the DAG gets a paired verification callable.
When the agent completes a task, the verify function runs automatically.
Failures trigger replan/retry with failure context.
**Implementation:**
- Add `verify_command: str | None` field to TaskRecord
- TaskGenerator assigns verify commands from project's test/build commands
- Orchestrator runs verify after each task completion (before marking COMPLETED)
- Failure feeds back into memory as regression + triggers re-queue
**Effort:** 2 days

### 3. Q-Learning Router
**Source:** Ruflo agent orchestration framework
**What:** Track success rates per (agent_type, task_category) pair.
Update Q-values after each task. Dispatch to highest-scoring agent.
Replaces hardcoded routing strengths with learned preferences.
**Implementation:**
- New `runner_scores` table: (runner_type, task_type, wins, losses, q_value)
- After each task: update win/loss count, recalculate Q = wins/(wins+losses)
- Scheduler uses Q-values instead of static routing config
- Bootstrap from current policies.yaml routing as initial priors
**Effort:** 1-2 days

### 4. Output-Hash Loop Detection
**Source:** OpenClaw watchdog pattern
**What:** Current loop detector checks for repeated lines. Upgrade to
fingerprint (tool_name, args_hash, output_hash) tuples in a sliding window.
Catches agents that rephrase the same failing approach differently.
**Effort:** 1 day

### 5. Tests-First Task Template
**Source:** Medium (Lanham, 2026)
**What:** Before dispatching an implementation task, first dispatch a
"write acceptance tests" task. The implementation task's verification =
"tests pass." Reduces thrash by ~50%.
**Implementation:**
- New task template: `build_test_first_task(project)` that creates 2 chained tasks
- Task 1: "Write failing tests for X" (blocked_by: nothing)
- Task 2: "Implement X to make tests pass" (blocked_by: task 1)
**Effort:** 1 day

## Priority Tier 2 — High Value (v2.2)

### 6. Claude Code Hooks Integration
**Source:** Claude Code lifecycle hooks (21 events)
**What:** Instead of parsing Claude's stdout, hook into PreToolUse/PostToolUse/Stop
to get structured JSON of every action. Inject verification gates, stream telemetry.
**Challenge:** Requires Claude Code to be configured with hooks pointing to Overmind.
**Effort:** 2-3 days

### 7. Sleep-Time Compute (Pre-Computation)
**Source:** Letta/MemGPT
**What:** Between sessions, Overmind pre-computes likely next tasks, pre-digests
project context, and caches results. When user starts, serve pre-computed context.
Cuts live token budgets by ~5x.
**Effort:** 2 days

### 8. Counterfactual Dreaming
**Source:** Active Dreaming Memory (ADM) research
**What:** During dream consolidation, generate counterfactuals: "Task X failed
because agent chose approach A. Approach B would have succeeded because [reason]."
Store as negative examples. Stronger than simple summarization.
**Effort:** 2 days

### 9. Codebase Map Injection
**Source:** Codex Orchestrator (kingbootoshi)
**What:** Pre-generate architectural context map per project. Inject into agent
prompts so agents skip exploration phase. Saves significant tokens.
**Effort:** 2-3 days

### 10. Self-Improving Prompt Loop
**Source:** AutoAgent (kevinrgu/autoagent)
**What:** Hill-climb on verification pass rates. Keep prompt changes that improve
results, discard those that don't. Prompts evolve over time.
**Effort:** 2-3 days (builds on #1)

## Priority Tier 3 — Future (v3.0)

### 11. TripleShot Competing Agents
**Source:** Claudio
**What:** For critical/high-risk tasks, spawn 3 competing agents.
A judge agent selects the best output. Triples cost but maximizes quality.

### 12. SQLite Mailbox
**Source:** Overstory
**What:** Agents communicate via shared SQLite DB instead of pipes.
Enables true multi-agent collaboration on complex tasks.

### 13. Dashboard / API
**Source:** Praktor
**What:** Web UI for monitoring agent fleet, memory, task queue, verification results.

### 14. erans/autoagent-action Patterns
**Source:** GitHub Action that dispatches to Claude/Codex/Gemini in CI workflows.
**What:** Steal the CI integration pattern for automated PR verification.

## Portfolio-Specific Insights

From the audit of 312 projects:
- 122 meta-analysis projects should get `meta_analysis` routing preference
- 41 R-backed projects need R-specific test discovery (testthat, Rscript)
- 44 extreme-rigor projects should always get the highest verification burden
- Browser apps (175) dominate — browser test infrastructure is critical
- Only 2 projects have Playwright — opportunity to add browser test coverage
