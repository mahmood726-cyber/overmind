# Verifying Overmind / TruthCert signed bundles

This project signs its verification bundles so **anyone can confirm a result is
authentic and untampered — without a shared secret and without trusting the
author's machine.** Three tiers, in increasing strength; you never need a key to
*publish*, only the author needs one to *sign*.

## The tiers

| Tier | Method | What a publisher needs | What a verifier needs | Property |
|---|---|---|---|---|
| 0 | SHA-256 content seal | nothing | nothing (recompute the hash) | tamper-evident integrity |
| 1 | **Ed25519** (default) | a private key (one-time, local) | this repo's public key | integrity **+ authorship**, offline |
| 2 | Sigstore + Rekor | an OIDC identity (online) | sigstore trust root | the above **+ public transparency log** |

**Why not HMAC anymore.** HMAC is symmetric: the same secret signs *and* verifies,
so any verifier could forge. It is retained only so previously-archived
HMAC bundles stay verifiable; all new bundles default to Ed25519 (asymmetric),
where the verifier only ever holds the **public** key and cannot forge.

## Verify an Ed25519 bundle (the default)

A signed bundle records `signature_method: "ed25519"`, a base64 `signature`, and
the base64 `public_key` it was signed with. Two checks:

**1. Integrity + signature** (no env, no secret):

```python
from overmind.verification.signers import SignResult, verify_result
res = SignResult(method="ed25519", signature=BUNDLE["signature"],
                 public_key=BUNDLE["public_key"])
assert verify_result(CANONICAL_PAYLOAD_BYTES, res)   # True iff untampered
```

**2. Authorship** — confirm the bundle was signed by *this project*, not someone
who re-signed a tampered copy with their own keypair. Pin the embedded key
against the canonical public key committed here:

```
BUNDLE["public_key"]  ==  the Key: line in OVERMIND_SIGNING_PUBKEY.txt
```

If both pass, the bundle is authentic and unmodified. Step 2 is the important
one: step 1 alone only proves *internal* consistency.

## Verify a Tier-0 content seal (keyless artifacts)

Browser-exported artifacts (e.g. allmeta TruthCert receipts) carry a keyless
`contentHash` instead of a signature when no signing key is configured —
publishing is never blocked. Recompute SHA-256 over the canonical content (the
manifest with its own `contentHash` field removed) and compare. This proves the
content has not changed; it makes **no authorship claim** (anyone can produce a
valid hash), which is stated honestly in the artifact.

## Keys

- Public key: `OVERMIND_SIGNING_PUBKEY.txt` (safe to share; verifies only).
- Private key: generated by `Ed25519Signer.generate_keypair()`, lives outside the
  repo (`~/.overmind/keys/`), gitignored, referenced via `OVERMIND_ED25519_KEY`.
- Rotation: generate a new keypair, append the new public key here with its date,
  and keep the old one listed so historical bundles still verify.
